# KS2 Mastery Hardening Pass 1 — Known-Faults Baseline

This document is a signed snapshot of known faults at the start of the hardening sprint. Every subsequent unit (U2 through U13) references this baseline as its "was this already broken?" oracle. Entries without a `(tracked in U?)` suffix are flagged as unaddressed and either picked up in follow-up plans or accepted as residual risk.

Entry format: one-line description + `(tracked in U?)` suffix when a hardening unit covers it.

## Visual faults

- Card clipping on narrow mobile viewports (360 px and below) in several learner hub surfaces, with content truncated rather than wrapping cleanly. (tracked in U5, U12)
- Horizontal overflow on learner activity rows at 360 px viewport when item titles exceed container width. (tracked in U12; **practice-surface overflow portion landed** via `tests/playwright/spelling-golden-path.playwright.test.mjs` U12 scene asserting `document.documentElement.scrollWidth <= clientWidth` on mobile-390 with a live spelling session.)
- Inconsistent spacing between session summary cards and adjacent reward tiles, causing uneven vertical rhythm on parent hub. (tracked in U12)
- Broken card states when session data is partially loaded, showing skeleton-card outline without the shimmer animation. (tracked in U12)
- Modal-body scroll trap on very tall content where the backdrop scrolls instead of the modal interior on some mobile browsers. (tracked in U5, U12)
- Toast overlap when multiple toasts fire within a short window, stacking visually rather than queueing. (tracked in U5, U12; **toast-vs-submit layering portion landed** via `tests/toast-positioning-contract.test.js` parser-level z-index + position + keyframe-allowlist + reduced-motion invariants AND `tests/playwright/spelling-golden-path.playwright.test.mjs` U12 scene asserting toast bounding box does not overlap submit button on mobile-390.)
- Monster / effect sprite layering glitch where celebration sprites render behind the learner's current answer card on specific viewport widths. (tracked in U5, U12; **toast-shelf-vs-celebration z-index contract landed** via `tests/toast-positioning-contract.test.js` asserting `.monster-celebration-overlay` z-index strictly exceeds `.toast-shelf`. Remaining sprite-layering cases are still open.)
- Tiny mobile glitches: button-tap highlight persisting after navigation, focus ring flashing on route change. (tracked in U12; **route-change audio-cleanup portion landed** via `tests/route-change-audio-cleanup.test.js` pinning the `tts.stop()` side-effect contract on every adult-surface route entry — `navigate-home`, `open-subject`, `open-codex`, `open-parent-hub`, `open-admin-hub`, `open-profile-settings`, `learner-select` — so mid-prompt transitions stop audio within 100ms.)
- Dark-mode and low-contrast colour pairings on destructive-action buttons and secondary labels, not meeting WCAG 2.2 AA contrast ratio. (tracked in U12)
- Inconsistent empty-state copy and illustration usage across `WordBank`, `ActivityFeed`, and `RewardShelf` surfaces. (tracked in U12; **button-label drift portion landed** via `tests/button-label-consistency.test.js` asserting canonical labels (Start / Continue / Try again / Finish / Back to dashboard) keep their exact spelling across `src/surfaces` + `src/subjects` + `src/app` and classifying every non-canonical label into either a `LABELS_TO_NORMALISE` list or a `LABELS_NOT_BLOCKING` allowlist — any new bespoke label fails the test until triaged.)

## Runtime faults

- Intermittent console errors from third-party library warnings that have not been triaged and suppressed. (tracked in U5, U9)
- Unhandled promise rejections observed in chaos scenarios (401 during refresh, 409 during retry burst). (tracked in U9; **landed** via `tests/playwright/chaos-http-boundary.playwright.test.mjs` 401-unauth + 409-stale-write + 409-idempotency-reuse scenes backed by `tests/helpers/fault-injection.mjs` named-export `__ks2_injectFault_TESTS_ONLY__` and the per-request opt-in header in `tests/helpers/browser-app-server.js`.)
- Double-submit on subject-command buttons when learners double-tap on touch devices, currently debounced by adapter retry but not by the UI. (tracked in U9, U12; U12 destructive-action contract confirms all three danger-zone commands (`learner-delete`, `learner-reset-progress`, `platform-reset-all`) route through `ports.confirm(...)` BEFORE any destructive side effect runs — `tests/destructive-action-confirm-contract.test.js` pins the contract for both confirm-yes and confirm-no paths. The remaining non-destructive double-tap debouncing is still open.)
- Switching learner mid-session leaves stale queue state in memory for a brief window before bootstrap rehydrates. (tracked in U9, U10; U9 portion covered by chaos refresh-during-submit scene asserting pending-queue rehydration via `localStorage` inspection. **U10 portion landed** via `tests/playwright/multi-tab-bootstrap.playwright.test.mjs` exercising three `browser.newPage()` tabs inside one `browser.newContext()` — shared `localStorage` confirms the `bootstrapCoordinationStorageKey` lease arbitrates writes across tabs, with an explicit cross-context isolation guard scene documenting why `browser.newContext()`-per-tab is the wrong primitive, plus a leader-leaves liveness scene.)
- Back-button behaviour after session completion returns to an intermediate state rather than the hub. (tracked in U9, U12)
- Stale state after refresh in rare multi-tab scenarios where the coordination lease arbitrates writes but the shell's local cache is from a previous tab. (tracked in U9, U10; U9 portion covered by chaos refresh-during-submit scene; **U10 multi-tab coverage landed** via `tests/playwright/multi-tab-bootstrap.playwright.test.mjs` — three shared-context pages coordinate to a bounded bootstrap-request count; lease visible via `page.evaluate(() => localStorage.getItem(...))` against `ks2-platform-v2.api-cache-state:<scope>:bootstrap-coordination`.)
- Demo session expiry surfaces as a generic error toast rather than a bespoke expiry banner with convert-to-real-account call-to-action. (tracked in U9, U12; U9 portion covered by the 401-unauth chaos scene proving the shell survives the auth failure without crash.)
- Degraded-sync banner occasionally flashes on fresh bootstraps when the initial `persistence` channel snapshot fires before steady-state. (tracked in U9; **landed** via U9 `data-testid="persistence-banner"` + chaos control scene asserting the banner is absent when no faults fire.)
- TTS failures (slow prompt-token resolution, 500 from prompt endpoint) degrade silently without retry transparency or a "tap to replay" affordance. (tracked in U9, U12; U9 portion covered by the slow-TTS chaos scene asserting practice surface stays reachable.)
